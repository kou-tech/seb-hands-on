# GitHubでコードを公開しよう

作成したアプリのコードを、GitHubで管理・公開しましょう。

## 作業時間の目安

約30分

---

## Gitとは？GitHubとは？

### Git（ギット）

**Git** は、プログラムのソースコードを保存し、変更履歴を記録・追跡することができるツールです。

例えば
- 昨日の状態に戻したい
- 誰がいつ何を変更したか知りたい
- 複数人で同じコードを編集したい

こんなときにGitが役立ちます。

### GitHub（ギットハブ）

**GitHub** は、Gitで管理したコードをオンラインで保存・共有できるプラットフォームです。無料で使えます。

多くのプログラマーがGitHubを使って、コードを公開したり共同開発したりしています。

**コーチより** GitとGitHubの違いを説明しましょう。Gitは「ツール」、GitHubは「サービス」です。Wordに例えると、Gitがアプリ本体で、GitHubがクラウドストレージのようなものです。

## Step 1 Gitの確認

### 1-1. Gitがインストールされているか確認

ターミナルで以下を実行

```bash
git --version
```

バージョン番号が表示されればOKです。

```
git version 2.x.x
```

表示されない場合は、Gitをインストールしてください。

**macOS:**
```bash
brew install git
```

**Windows:**
[https://git-scm.com/download/win](https://git-scm.com/download/win) からダウンロードしてインストール

---

## Step 2 Gitの初期設定

### 2-1. 名前とメールアドレスを設定

Gitに自分の名前とメールアドレスを登録します。

```bash
git config --global user.name "あなたの名前"
git config --global user.email "your-email@example.com"
```

例：
```bash
git config --global user.name "Hanako Yamada"
git config --global user.email "hanako@example.com"
```

**⚠️ 重要**: ここで設定した名前とメールアドレスは、他の人にも公開されます。本名を使いたくない場合は、ニックネームを使ってください。

### 2-2. 設定を確認

```bash
git config --global user.name
git config --global user.email
```

設定した名前とメールアドレスが表示されればOKです。

---

## Step 3: ローカルリポジトリを作成

プロジェクトをGitで管理できるようにします。

### 3-1. プロジェクトフォルダに移動

```bash
cd ~/Desktop/diary-app
```

（作成した場所に合わせて変更してください）

### 3-2. Gitリポジトリを初期化

```bash
git init
```

以下のようなメッセージが表示されます。

```
Initialized empty Git repository in ...
```

**コーチより** `git init` でカレントディレクトリがGitリポジトリになりました。`.git` フォルダが作成され、ここに変更履歴が保存されます。

### 3-3. ファイルをステージングに追加

```bash
git add .
```

`.` は「すべてのファイル」を意味します。

### 3-4. コミット（保存）

```bash
git commit -m "Initial commit: 日記アプリの作成"
```

`-m` の後ろに、変更内容のメッセージを書きます。

**コーチより** コミットは「スナップショットを撮る」イメージです。この時点の状態を記録して、いつでも戻れるようになります。

---

## Step 4 GitHubアカウントを作成

### 4-1. GitHubにアクセス

[https://github.com/](https://github.com/) を開きます。

### 4-2. サインアップ

1. 「Sign up」をクリック
2. メールアドレス、パスワード、ユーザー名を入力
3. 認証を完了してアカウント作成

**ヒント**: ユーザー名は、GitHubのURLに使われます（`https://github.com/your-username`）

---

## Step 5 GitHubに新しいリポジトリを作成

### 5-1. 新規リポジトリを作成

1. GitHubにログイン
2. 右上の「+」マーク → 「New repository」をクリック
3. 以下を入力
   - **Repository name** `diary-app`
   - **Description** 「Laravel製の日記アプリ」（任意）
   - **Public** を選択（誰でも見られる）
   - その他はチェックを入れない
4. 「Create repository」をクリック

---

## Step 6 ローカルとGitHubを連携

### 6-1. リモートリポジトリを追加

GitHubに表示されているコマンドを参考に、以下を実行します。

```bash
git remote add origin https://github.com/あなたのユーザー名/diary-app.git
```

例：
```bash
git remote add origin https://github.com/hanako-yamada/diary-app.git
```

### 6-2. ブランチ名を変更（必要な場合）

```bash
git branch -M main
```

### 6-3. GitHubにプッシュ（アップロード）

```bash
git push -u origin main
```

初回実行時は、GitHubへのログインが求められます。

**macOS** ユーザー名とパスワード、または個人アクセストークンを入力

**Windows** ブラウザが開き、認証画面が表示されます

---

## Step 7 GitHubで確認

ブラウザでGitHubのリポジトリページを開きます。

`https://github.com/あなたのユーザー名/diary-app`

ファイル一覧が表示されていれば成功です！

---

## 変更を保存してGitHubに反映する

今後、コードを変更したときは、以下の流れで保存・アップロードします。

### 1. 変更を確認

```bash
git status
```

変更されたファイルが表示されます。

### 2. ステージングに追加

```bash
git add .
```

### 3. コミット

```bash
git commit -m "変更内容の説明"
```

例：
```bash
git commit -m "日記の削除機能を改善"
```

### 4. GitHubにプッシュ

```bash
git push
```

これで、GitHubに反映されます！

## まとめ

### 学んだコマンド

| コマンド | 説明 |
|---------|------|
| `git init` | Gitリポジトリを初期化 |
| `git add .` | すべてのファイルをステージング |
| `git commit -m "message"` | 変更を保存 |
| `git push` | GitHubにアップロード |
| `git status` | 変更状態を確認 |

### Gitのワークフロー

```
ファイルを変更
    ↓
git add .（ステージング）
    ↓
git commit -m "message"（保存）
    ↓
git push（GitHubにアップロード）
```

## よくある質問

### Q: `.env` ファイルがGitHubに公開されませんか？

A: 大丈夫です。Laravelは `.gitignore` ファイルで `.env` を除外しているので、GitHubにはアップロードされません。

### Q: パスワードを毎回入力するのが面倒です

A: SSH鍵を設定すると、パスワード入力が不要になります。コーチに相談してください。

### Q: コミットメッセージは日本語でもいいですか？

A: はい。ただし、チームで開発する場合は英語が推奨されることもあります。

**次のステップ** アプリをさらに改良したい方は、発展課題に挑戦してみましょう！

**トラブルシューティング**

- **git push でエラーが出る**: 認証情報を確認してください。個人アクセストークンが必要な場合があります
- **コミットできない**: ファイルが保存されているか確認してください
- **リモートURLが間違っている**: `git remote -v` で確認できます
